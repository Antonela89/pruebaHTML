Estilo HTML

  • Declarar siempre el tipo de documento
    • !DOCTYPE html
  • Usar minúsculas para nombrar los elementos
    • Las minúsculas se ven más limpias y son más fáciles de escribir
    • Tipo de letra usado por los desarrolladores
  • Cerrar todos los elementos HTML
  • Citar los valores de los atributos
  • En las imagenes siempre colocar alt, width y height
  • No colocar espacios alrededor de los signos iguales
  • Evitar líneas de código largas
  • Agregar sangrias de ser necesario
    • Para facilitar la lectura:
      • Agregar líneas de espacios en blanco para separar bloques de código
      • Agregar 2 espacios de sangria
      • No usar la tecla Tab
    • No omitir el title
    • No omitir etiquetas Html, Head y Body
    • Agregar el atributo lang
    • Definir siempre meta charset
    • Definir la configuración de la ventana gráfica: diseño responsivo
    • Escritura de comentarios
      • Los comentarios cortos deben escribirse en una línea
        • <!-- This is a comment -->
      • Los comentarios largos son más fáciles de observar si tienen una sangria con dos espacios
        • <> 
                          <!--
                    This is a long comment example. This is a long comment example.
                    This is a long comment example. This is a long comment example.
                    -->
